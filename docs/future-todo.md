# Future TODO

> 本文档只记录未来可能实现的想法，不是当前项目需求、实施计划、验收标准或决策依据。
>
> 默认情况下，进行项目开发、排错、评审或规划时不要读取、引用或据此推断任务范围。只有当用户明确说要调用、参考或处理这份 TODO 文档时，才将其中内容纳入当前任务。

状态：未拆解，未排期。

## 1. 更贴合浏览器 AI 使用

- 让 Craft 底层能力更贴合浏览器 AI 的使用场景。
- 考虑专门提供一套 CLI 适配，方便快速测试浏览器功能。
- 让这套浏览器使用能力也能被 GUI 扩展使用。

## 2. 会话编号、跨会话查看与引用

- 为每个会话提供一个简单编号。这里暂按上下文将原话中的“绘画”理解为“会话”，实现时再确认术语。
- 给模型提供快速查看其他会话内容的工具或其他等效方式。
- 允许把侧栏中的会话卡片拖入对话框，作为对其他会话内容的引用。
- 支持针对其他会话设置定时任务，例如某个会话完成后触发后续任务。

## 3. 语音输入

- 增加语音输入功能。

## 4. 工作模式隔离

- 增加模式功能，用于隔离不同的工作模式。
- 模式可能需要隔离系统提示词、上下文、工具、skill、子智能体等内容。
- 考虑扩展隔离，以及每种模式自定义 UI 样式的能力。

## 5. Agent / 智能体设置模块

- 在设置中增加 Agent 或“智能体”模块。
- 支持修改系统默认提示词。
- 支持配置系统可见的工具。
- 支持预设子智能体。
- 支持修改压缩上下文时使用的提示词。

## 6. 额外内容面板与右侧查看区

- 恢复额外内容面板相关功能。
- 考虑把其他查看类功能放入右侧面板。

## 7. GUI 能力优先于斜杠命令

- 对已经在 Craft GUI 中实现的能力，在用户输入 `/` 时屏蔽对应的斜杠命令，不再重复提供两套操作入口。
- 只有尚未提供 GUI 入口的能力，才继续通过斜杠命令操作。
